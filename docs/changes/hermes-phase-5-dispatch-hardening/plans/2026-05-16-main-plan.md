### Task D3 â€” `CostCeilingMonitor` service

**Files:** `packages/orchestrator/src/cost/cost-ceiling-monitor.ts` (new)

**Changes:**

- `class CostCeilingMonitor`
- `registerTask(taskId, ceiling)`: starts tracking
- `recordTurn(taskId, usage, modelName)`: applies pricing, accumulates, fires abort signal when exceeded
- `unregisterTask(taskId)`: stops tracking, returns final `costUsd`
- Emits `EventEmitter` events: `'abort'` (with `{ taskId, costUsd }`), `'warn'` (with `{ taskId, costUsd, pctOfCeiling }`)

**Verification:**

- Unit tests (next task) exercise the state machine

### Task D4 â€” `CostCeilingMonitor` unit tests

**Files:** `packages/orchestrator/test/cost/cost-ceiling-monitor.test.ts` (new)

**Cases:**

- Task without a ceiling never emits `abort` or `warn`
- Cumulative cost exceeding ceiling emits `abort` exactly once
- Warn threshold (e.g., `warnAtPct: 80`) fires before abort
- `unregisterTask` returns the cumulative cost
- Two tasks tracked simultaneously do not interfere

**Verification:**

- `pnpm --filter @harness-engineering/orchestrator test` green

### Task D5 â€” Scheduler integration

**Files:** `packages/orchestrator/src/maintenance/scheduler.ts` (existing)

**Changes:**

- Before dispatch: `monitor.registerTask(taskId, definition.costCeiling)`
- Subscribe to `'abort'` event before starting the agent runner
- On abort: signal the runner to stop, set `RunResult.status = 'failure'`, `error = 'cost_ceiling_exceeded'`, `costUsd = monitor.unregisterTask(taskId)`
- On success / other failure: `costUsd = monitor.unregisterTask(taskId)`

**Verification:**

- Integration test (next task) green
